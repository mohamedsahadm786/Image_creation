# Qwen-tuned prompt experiment — A/B/C comparison

Sibling experiment to the parent `step2_qwen_edit_2511/` swap. This one keeps
the Qwen image-edit model the same (`fal-ai/qwen-image-edit-2511`) but
**regenerates the Step 2 prompt with a Qwen-tuned master prompt** and compares
its output against:

- **A — Nano Banana** (existing baseline image, reused from a Plan A run)
- **B — Qwen with OLD NB-shaped prompt** (existing image, reused from the
  previous `step2_qwen_edit_2511/batch_runner/outputs/` batch)
- **C — Qwen with NEW Qwen-tuned prompt** (NEW — generated by this experiment)

## Why this experiment exists

The first Qwen batch produced concrete, reproducible failure modes vs the
Nano Banana baseline:

1. **Mirrored product** — TIRZEPATIDE / ALLUVI text rendered backwards
2. **Pose drift** — arm rotated to a different position than specified
   (e.g. straight up over head when prompt said "at face level")
3. **Identity-lock language softened** — face/hair/outfit/scene mostly
   survived but the holding pose was freely reinterpreted
4. **Rare anatomy artifacts** — three arms, six or seven fingers per hand
   (low frequency but high visibility when it occurs)

These are documented Qwen-Image-Edit-2511 sensitivities. The
NB-shaped master prompt — which was correct for Nano Banana's unified Gemini
transformer — does not address them. The Qwen-tuned master prompt
(`master_prompt_step2_qwen.md`) does, by adding three Qwen-specific clauses
on top of every existing rule from the original master prompt.

## What changed in the master prompt

`master_prompt_step2_qwen.md` is a sibling to the project-root
`prompts/master_prompt_step2.md`. **Every rule from the original is preserved
verbatim** — the 8 operating principles, all four sentence-level structures,
the white-base preservation, the scale anchor, the hand-visibility rule, the
packaging-description bans, the persona-alteration bans, the color-on-product
bans, both calibration examples, the full compliance check schema, and the
JSON output contract.

Three Qwen-specific additions:

1. **Principle 1 reversed.** The original bans "Image 1" / "Image 2"
   syntax (correct for Nano Banana). The Qwen variant *requires* positional
   reference syntax — "the person from the first image", "the product from
   the second image" — because Qwen's official guidance recommends it.

2. **Principle 9 added.** Three new mandatory clauses addressing observed
   Qwen failure modes:
   - **9.a — Product orientation lock** (anti-mirroring clause embedded in
     Sentence 2)
   - **9.b — Position re-anchoring** (positive + negative position
     constraints, e.g. "at face level — at face level specifically, not
     above her head, not at her chest")
   - **9.c — Anatomy sanity** (exactly two arms, two hands, five fingers
     each — embedded in Sentence 3)

3. **Word budget bumped** from 150–200 to **280–380**. The new anchor
   clauses, role-tags, orientation lock, position re-anchoring, and anatomy
   sanity add ~80–130 words; the prompt is *more* specified, not less.
   Qwen's documented preference is for detailed, specific prompts.

Two anti-examples added (F: missing orientation lock, G: missing anatomy
clause, H: missing position re-anchoring) and Anti-Example A
**reversed** — the generic "the persona reference photo" / "the product
reference photo" syntax is now BANNED in the Qwen variant.

## Cost & timing

Per scenario:
- ~$0.18 — Opus 4.7 prompt regeneration (~8K input + ~800 output tokens)
- ~$0.04 — Qwen API call
- **~$0.22 total per scenario**
- ~5s Opus + ~30s Qwen = ~35s wall time per scenario

For the full 30-scenario batch: **~$6.60 and ~17–18 minutes** sequential.

## Quick start

### 1. Sanity check on a single scenario first (~$0.22)

Pick a scenario that produced a clear Qwen failure in the previous batch.
`pilates_mat_morning_handheld_09` is a known-difficult one:

```powershell
python -m experiments.step2_qwen_edit_2511.qwen_tuned_prompt.run `
    --reuse-run outputs/runs/2026-05-08_17-04-15_plan_a/pilates_mat_morning_handheld_09 `
    --reuse-qwen-v1 experiments/step2_qwen_edit_2511/batch_runner/outputs/2026-05-08_20-09-23_batch/pilates_mat_morning_handheld_09
```

The `--reuse-qwen-v1` argument is **optional** — without it the v1 panel
shows "not provided" and you'll only see NB vs Qwen-v2 (no v1 middle column).
Strongly recommended you pass it for the full A/B/C experience.

Output:
```
experiments/step2_qwen_edit_2511/qwen_tuned_prompt/outputs/<timestamp>_<scenario_id>/
    chain.html                          ← open this — 5-panel A/B/C drill-down
    01_scenario.yaml                    (copied)
    02_step1_prompt.json                (copied)
    03_step1_persona.jpg                (copied)
    04_step2_nb_prompt.json             (copied — OLD prompt for diff)
    04_step2_qwen_prompt.json           (NEW — Qwen-tuned prompt)
    05_step2_nb.jpg                     (copied)
    05_step2_nb_meta.json               (copied)
    05_step2_qwen_v1.jpg                (copied if --reuse-qwen-v1 given)
    05_step2_qwen_v1_meta.json          (copied if --reuse-qwen-v1 given)
```

## Inspecting results

The chain.html shows:

| Panel | What |
|-------|------|
| Step 0 | Source persona.jpg |
| Step 1 — PuLID | Persona scene (no product) |
| Step 2a — Nano Banana | NB with NB-shaped prompt (existing baseline) |
| Step 2b — Qwen v1 | Qwen with OLD NB-shaped prompt (previous batch) |
| Step 2c — Qwen v2 | Qwen with NEW Qwen-tuned prompt (this experiment, **purple-bordered**) |

Below the panels: the OLD prompt and NEW prompt side-by-side, so you can
read exactly what differs. Also: per-image metadata (elapsed, cost, seed).

Things to look for in the Qwen-v2 column vs Qwen-v1:

- ✓ Product text reads forward (TIRZEPATIDE / ALLUVI not mirrored)
- ✓ Holding position matches the prompt spec (not drifted)
- ✓ Both arms/hands have correct anatomy (no extras, no fused fingers)
- ✓ Identity (face, hair, outfit, scene) preserved as well as v1 or better
- ✓ White product base preserved (no amber tint)

## Decision tree after the batch

| Result | Next step |
|--------|-----------|
| Qwen v2 fixes the orientation/pose/anatomy issues without regressing identity | Adopt Qwen + Qwen-tuned prompt as primary. Deprecate Plan A NB. |
| Qwen v2 fixes some but not all | Iterate the master prompt — paste failed outputs back, add per-archetype calibration examples |
| Qwen v2 fixes nothing or regresses | Fall back to FLUX-2-Klein-9B experiment (already scaffolded, just run the batch) or Backup #2 Seedream 4.5 |
| Qwen v2 produces wildly different identities | Issue is not the prompt — likely Qwen's identity preservation ceiling. Phase 3 fine-tuning becomes the path. |
